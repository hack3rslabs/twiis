Security Statement

Our internal security posture reflects the same standards we demand for our clients. Last updated: April 2026.

1. Our Internal Security Commitment

Twiis Innovations practices what it preaches. Our internal infrastructure, development processes, and operational procedures are held to the same elite security standards we implement for enterprise clients. We operate on a zero-trust model internally and conduct regular self-assessments of our own security posture.

2. Data Security Practices

  • All client data is encrypted in transit using TLS 1.3+
  • Data at rest is encrypted using AES-256 encryption
  • Access to client systems follows least-privilege principles
  • Multi-factor authentication (MFA) is mandatory for all team members
  • Security tools and access credentials are rotated regularly

3. Secure Development Practices

All software and tools developed by Twiis undergo mandatory code review, SAST scanning, and security testing before deployment. We follow the OWASP Secure Development Lifecycle and maintain a secure software development policy that governs all internal tooling.

4. Personnel Security

All Twiis team members undergo background verification before being granted access to client environments. Security awareness training is conducted quarterly, and all staff sign comprehensive non-disclosure agreements and acceptable use policies.

5. Incident Response

Twiis maintains a documented Incident Response Plan (IRP). In the event of a security incident affecting client data, we commit to notifying affected clients within 72 hours of discovery, in accordance with GDPR breach notification requirements.

6. Responsible Disclosure

If you believe you have discovered a security vulnerability in Twiis's systems or website, please disclose it responsibly by emailing help@twiis.in with a detailed description. We commit to acknowledging all responsible disclosures within 48 hours and resolving confirmed vulnerabilities within 30 days.

7. Physical Security

Twiis operates in secure environments with restricted physical access. All sensitive work involving client data is conducted in controlled environments with appropriate physical security controls.

8. Third-Party Risk Management

Any third-party services used in service delivery undergo security vetting before engagement. We maintain a register of third-party processors and conduct annual reviews of their security postures.
